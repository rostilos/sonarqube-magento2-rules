Why is this an issue?

Magento 2 promotes a clear separation of concerns between business logic and presentation. Using $this in templates creates tight coupling between templates and blocks, making code harder to test and maintain. Similarly, using generic helper variables ($helper or $_helper) often leads to complex, overloaded templates that are difficult to understand and maintain.

Noncompliant Code Example

$this->escapeHtml($this->getProduct()->getName());
$helper->formatPrice($product->getPrice())

Compliant Solution

$viewModel->getProductName();
$block->getFormattedPrice();
