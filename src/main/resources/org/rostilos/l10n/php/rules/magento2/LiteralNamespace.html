Why is this an issue?

In Magento 2, using hardcoded (literal) namespace strings in PHP code can reduce flexibility and maintainability. Instead, dynamic approaches such ::class should be used to improve code robustness, prevent typos, and facilitate refactoring.

Noncompliant code example

    public function notOk()
    {
        $this->create('Magento\CustomerSegment\Model\Segment\Condition\Customer\Address'); // Noncompliant
    }

Compliant example

  public function ok()
    {
        $this->create(Magento\CustomerSegment\Model\Segment\Condition\Customer\Address::class); // Compliant
    }
