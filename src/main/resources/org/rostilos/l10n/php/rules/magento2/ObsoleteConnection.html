Why is this an issue?

This rule detects usage of obsolete connection methods that are no longer recommended in Magento development.

Noncompliant Code Example

class MyClass
{
    public function doSomething()
    {
        $connection = $this->_getReadConnection();
        $connection = $this->getWriteAdapter();
    }
}

Compliant Solution

class MyClass
{
    public function doSomething()
    {
        $connection = $this->getConnection();
    }

}

Why is this important?

Magento has deprecated several connection-related methods in favor of the more unified getConnection() approach. Using the newer methods:

  • Ensures compatibility with future versions
  • Follows Magento best practices for database connections
  • Makes code more maintainable and consistent
  • Prevents potential issues as obsolete methods may be removed in future versions

Obsolete Methods

The following methods are considered obsolete:

  • _getReadConnection
  • _getWriteConnection
  • _getReadAdapter
  • _getWriteAdapter
  • getReadConnection
  • getWriteConnection
  • getReadAdapter
  • getWriteAdapter
