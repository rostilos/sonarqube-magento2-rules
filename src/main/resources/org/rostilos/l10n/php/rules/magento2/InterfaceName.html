Why is this an issue?

This rule verifies that interface names follow the convention of ending with the 'Interface' suffix.

Why is this important?

Following naming conventions for interfaces provides several benefits:

  • Improves code readability by clearly identifying the purpose of the declaration
  • Makes it easier to distinguish interfaces from classes at a glance
  • Follows object-oriented best practices and established conventions
  • Maintains consistency across the codebase

Noncompliant code example

interface Customer {
    public function getName(): string;
    public function getEmail(): string;
}

Compliant example

interface CustomerInterface {
    public function getName(): string;
    public function getEmail(): string;
}

Exceptions

There are no exceptions to this rule. All interfaces should follow this naming convention.
