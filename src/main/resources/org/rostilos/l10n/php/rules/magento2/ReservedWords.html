Why is this an issue?

PHP has several reserved words that cannot be used to name a class, interface, trait, or in namespace parts. Using these words can lead to unexpected behavior or errors in PHP code.

Noncompliant Code Example

namespace App\String\Service; // Noncompliant - 'String' is a reserved word

class Int // Noncompliant - 'Int' is a reserved word
{
    // Class implementation
}

interface Iterable // Noncompliant - 'Iterable' is a reserved word
{
    // Interface definition
}

trait Bool // Noncompliant - 'Bool' is a reserved word
{
    // Trait implementation
}

Compliant Solution

namespace App\StringValue\Service; // Compliant - 'StringValue' instead of 'String'

class Integer // Compliant - 'Integer' instead of 'Int'
{
    // Class implementation
}

interface IterableInterface // Compliant - 'IterableInterface' instead of 'Iterable'
{
    // Interface definition
}

trait BooleanTrait // Compliant - 'BooleanTrait' instead of 'Bool'
{
    // Trait implementation
}

Why is this important?

PHP reserves certain words for current or future language features. Using these reserved words as class names or in namespaces:

  • May cause conflicts with PHP's internal types
  • Could lead to errors in current or future PHP versions
  • Creates confusion between user-defined types and built-in language constructs
  • May reduce code compatibility across different PHP versions

Reserved Words

The following words are considered reserved:

  • int (since PHP 7)
  • float (since PHP 7)
  • bool (since PHP 7)
  • string (since PHP 7)
  • true (since PHP 7)
  • false (since PHP 7)
  • null (since PHP 7)
  • void (since PHP 7.1)
  • iterable (since PHP 7.1)
  • resource (since PHP 7)
  • object (since PHP 7)
  • mixed (since PHP 7)
  • numeric (since PHP 7)
  • match (since PHP 8)
