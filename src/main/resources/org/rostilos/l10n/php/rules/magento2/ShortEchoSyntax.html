Why is this an issue?

This rule enforces the use of short echo syntax (<?= $var ?>) instead of regular echo statements (<?php echo $var; ?>) in .phtml template files.

Why is this an issue?

Using short echo syntax in template files has several benefits:

  • Improves readability by making template code more concise
  • Follows Magento 2 coding standards for template files
  • Reduces visual clutter in template files where many variables are output
  • Makes the code more maintainable by clearly distinguishing output statements from logic

Noncompliant Code Example

<div class="product-name">
    <?php echo $block->escapeHtml($_product->getName()); ?>
</div>

<span class="price">
    <?php echo $block->formatPrice($_product->getPrice()); ?>
</span>

<div class="description">
    <?php echo $block->escapeHtml($_product->getDescription()); ?>
</div>

Compliant Solution

<div class="product-name">
    <?= $block->escapeHtml($_product->getName()) ?>
</div>

<span class="price">
    <?= $block->formatPrice($_product->getPrice()) ?>
</span>

<div class="description">
    <?= $block->escapeHtml($_product->getDescription()) ?>
</div>

Exceptions

This rule applies only to .phtml template files. It does not enforce the short echo syntax in .php files where regular PHP code is written.
