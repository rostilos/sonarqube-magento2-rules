Why is this an issue?

This rule enforces that Magento 2 HTML templates must not directly instantiate objects. All required objects must be passed from Block objects instead.

In Magento's architecture, templates should only use objects that have been properly injected into and exposed by their associated Block classes. Direct object instantiation in templates violates Magento's dependency injection pattern, creates tight coupling, complicates testing, and reduces maintainability.

Noncompliant code example

$customer = new \Magento\Customer\Model\Customer(); // Noncompliant

Compliant solution

$customer = $this->getCustomer(); // Compliant
