Why is this an issue?

This rule detects possible improper usage of include and require functions in PHP.

Noncompliant Code Example

// Noncompliant: URL in include
include("https://example.com/script.php");

// Noncompliant: String concatenation
include($base_dir . "/file.php");

// Noncompliant: Variable usage
include($filename);

Why is this important?

File inclusion vulnerabilities occur when a web application dynamically includes external files based on user-controlled input. This can lead to:

  • Remote code execution vulnerabilities
  • Information disclosure vulnerabilities
  • Denial-of-service attacks

Using variables or string concatenation with include/require functions can be dangerous if user input is not properly sanitized.
